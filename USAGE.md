# Better-Student 使用说明

## 1. 安装

### 1.1 前置要求

- Python 3.8+
- pip 20.0+
- 麦克风设备
- 网络连接（用于API调用）

### 1.2 安装步骤

1. 克隆项目到本地
   ```bash
   git clone https://github.com/yourusername/Better-Student.git
   cd Better-Student
   ```

2. 安装依赖
   ```bash
   pip3 install -r requirements.txt
   ```

   > 注意：如果安装PyAudio失败，可能需要先安装PortAudio库：
   > - macOS: `brew install portaudio`
   > - Ubuntu: `sudo apt-get install portaudio19-dev`
   > - Windows: 下载并安装PortAudio Windows安装包

3. 配置环境变量
   ```bash
   # 复制环境变量模板文件
   cp .env.example .env
   # 编辑.env文件，填写千问API密钥等配置
   ```

4. 运行应用
   ```bash
   python src/main.py
   ```

## 2. 功能使用

### 2.1 语音识别

1. 在"语音识别"标签页中，点击"开始录音"按钮开始录音
2. 系统会实时将语音转换为文字并显示在文本区域
3. 识别到的专业术语会根据掌握状态显示不同的颜色：
   - 绿色：已掌握的术语
   - 蓝色：未掌握的术语
4. 鼠标悬停在术语上时，会显示术语的解释
5. 点击"停止录音"按钮停止录音

### 2.2 文本分析

1. 在"文本分析"标签页中，输入要分析的文本
2. 点击"分析"按钮进行分析
3. 系统会识别文本中的术语和问题，并显示在列表中
4. 点击列表中的术语或问题，会在下方显示详细信息

### 2.3 术语管理

1. 在"术语管理"标签页中，可以查看所有已存储的术语
2. 点击术语可以查看详细信息
3. 勾选"已掌握"复选框可以标记术语为已掌握
4. 点击"添加术语"按钮可以手动添加新术语
5. 点击"刷新"按钮可以刷新术语列表

### 2.4 设置

1. 在"设置"标签页中，可以配置：
   - API密钥（千问API和搜索API）
   - 音频设备和参数
   - 识别设置（超时时间、识别间隔）
2. 点击"保存设置"按钮保存配置
3. 点击"重置"按钮恢复默认设置

## 3. 功能特点

### 3.1 实时语音转文字
- 集成千问语音转文字模型，实现低延迟的实时文字输出
- 支持实时音频流处理，确保接近实时的文字输出

### 3.2 文本智能分析
- 专业术语识别：自动识别文本中的专业名词，提供准确的定义和解释
- 问题检测与处理：识别文本中教师提出的问题，自动检索上下文信息
- 联网搜索增强：针对检测到的问题，进行联网搜索获取相关信息
- AI问答整合：调用千问问答模型，结合上下文信息和搜索结果生成答案

### 3.3 术语掌握状态管理
- 存储专业术语，包含名词、出现位置、解释等信息
- 支持标记术语是否掌握
- 根据掌握状态显示不同颜色的术语标记
- 鼠标悬停显示术语解释

### 3.4 数据持久化
- 使用SQLite数据库进行本地存储
- 保存内容包含时间戳、完整文本内容等元数据
- 支持历史记录查询和导出

## 4. 技术实现

- **开发语言**：Python 3.8+
- **音频处理**：PyAudio
- **AI模型**：千问语音转文字API、千问问答API
- **数据存储**：SQLite
- **用户界面**：PyQt5
- **文本分析**：NLTK/Spacy
- **网络请求**：Requests

## 5. 故障排除

### 5.1 常见问题

1. **应用启动失败**
   - 检查Python版本是否为3.8+
   - 检查依赖是否安装完整
   - 检查环境变量是否配置正确

2. **语音识别失败**
   - 检查麦克风是否正常工作
   - 检查网络连接是否稳定
   - 检查千问API密钥是否正确配置

3. **术语识别不准确**
   - 确保Spacy中文模型已安装：`python -m spacy download zh_core_web_sm`
   - 尝试调整识别参数

4. **应用卡顿**
   - 关闭其他占用资源的程序
   - 调整识别间隔和超时时间

### 5.2 日志查看

应用的运行日志保存在`./logs/app.log`文件中，可以查看日志了解应用的运行状态和错误信息。
